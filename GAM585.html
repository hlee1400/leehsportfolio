3 Weeks

ProtoBuf Serialization

Project Overview

GLTF animation conversion + runtime control for fast loading and reliable playback

This project consists of two parts: an Animation Converter that imports GLTF models and exports animations into a custom binary format (ProtoBuf), and a Game Engine runtime that loads those files offline and plays animations with real-time controls.

The converter supports multiple animations from a single skeleton and produces compact, load-friendly data so runtime systems can avoid expensive parsing and reduce startup latency.

In-engine, I implemented playback controls including play / pause, stepping forward / backward, speed control (fast / slow), looping, and switching between animations—built to stay stable in both Debug and Release builds.

Runtime playback: playing 5 different clips at the same time, seperately

Key Features

Offline Animation Conversion

Converts GLTF animations into a custom ProtoBuf-based binary format for offline loading, enabling faster runtime startup and predictable data layout.

Multi-Clip Export From One Skeleton

Supports exporting multiple animation clips from a single skeleton, making it easy for the runtime to swap between clips without re-importing assets.

Runtime Playback Controls

Animation controller supports play, pause, loop, time stepping forward/backward, and speed scaling (fast/slow) for debugging and demos.

Memory-Safe, Test-Driven Development

Used custom memory tracking, applied RAII fundamentals, and ensured the system passes all provided tests + added custom edge-case tests.

Engineering Constraints & Highlights

Strict C++ Constraints (No STL)

Implemented required data handling without STL containers (no vector/map/etc.), using manual data structures and careful memory ownership to meet course rules and performance targets.

Robust Debug/Release Behavior

Worked through project-specific warnings and correctness checks to keep behavior consistent across builds, preventing “works in Debug only” regressions.

Version Control Discipline (Perforce)

Used frequent, small Perforce submits with clear changelist descriptions to maintain safe rollback points and isolate changes when debugging.
